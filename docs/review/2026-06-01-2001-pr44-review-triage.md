#### 2. Control-plane firewall is too narrow for intra-cluster traffic

Thread:

- `infra/terraform/hetzner-shared/main.tf`

Why valid:

- The control-plane firewall allows operator SSH, API server access, and ICMP.
- Worker nodes allow full internal TCP/UDP, but the control plane does not.
- k3s server nodes still need internal traffic from agents and cluster
  components.

Current implementation evidence:

- `infra/terraform/hetzner-shared/main.tf:37-59`
- `infra/terraform/hetzner-shared/main.tf:61-87`

Risk:

- broken overlay networking or incomplete cluster behavior
- possible metrics-server / kubelet reachability problems

Recommendation:

- allow internal cluster TCP/UDP from `var.network_cidr` on the control-plane
  firewall as well

Priority:

- `P1`

#### 3. PostgreSQL password is injected into SQL without escaping single quotes

Thread:

- `infra/terraform/modules/hetzner-k3s-cluster/templates/postgres-cloud-init.yaml.tftpl`

Why valid:

- The `DO $$ ... PASSWORD '${postgres_password}' ... $$;` command breaks if the
  password contains `'`.
- This is a real provisioning fragility for operator-supplied passwords.

Current implementation evidence:

- `infra/terraform/modules/hetzner-k3s-cluster/templates/postgres-cloud-init.yaml.tftpl:20`
- `infra/terraform/modules/hetzner-k3s-cluster/main.tf:112-114`

Risk:

- failed cluster bootstrap for otherwise valid passwords

Recommendation:

- escape `'` before rendering the template input

Priority:

- `P1`

Current implementation evidence:

- `P1`

#### 5. `terraform-hetzner.sh` always shifts args, which breaks env-driven usage

Thread:

- `scripts/terraform-hetzner.sh`

Current implementation evidence:

- `scripts/terraform-hetzner.sh:4-7`

  behavior

Recommendation:

Priority:

- `P1`
